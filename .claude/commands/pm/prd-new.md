# /pm:prd-new

Create a comprehensive Product Requirements Document through guided brainstorming and discovery.

## Usage
```
/pm:prd-new [feature-name]
```

## Process

You will guide the user through a structured discovery process to create a detailed PRD that includes:

1. **Vision & Objectives**
   - What problem are we solving?
   - Why is this important?
   - Success metrics

2. **User Stories & Use Cases**
   - Who are the primary users?
   - What tasks will they accomplish?
   - Edge cases and error scenarios

3. **Requirements**
   - Functional requirements
   - Non-functional requirements (performance, security, scalability)
   - Technical constraints

4. **Acceptance Criteria**
   - Clear, testable conditions for completion
   - Quality standards

5. **Dependencies & Risks**
   - External dependencies
   - Technical risks
   - Resource requirements

## Output

Save the PRD to `.claude/prds/{feature-name}.md` with full documentation for downstream planning.
